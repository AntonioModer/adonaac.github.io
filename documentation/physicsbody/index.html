-- physics enabled entity template
MyClass = fg.Class('MyClass', 'Entity')
MyClass:implement(fg.PhysicsBody)

function MyClass:new(area, x, y, settings)
    MyClass.super.new(self, area, x, y, settings)
    self:physicsBodyNew(area, x, y, settings)
end

function MyClass:update(dt)
    self:physicsBodyUpdate(dt)
end

function MyClass:draw()
    self:physicsBodyDraw()
end


Description

If you don’t know what mixins are, check this article I wrote about them.

The PhysicsBody mixin adds physics capabilities to an object. If you’re using the engine, then all your physics enabled entities should both inherit from Entity and have the PhysicsBody mixin implemented, like in the example above.

Methods

:physicsBodyNew(area[Area], x[number], y[number], settings[table[any][optional]])
  • area: a reference to the area where the entity is contained
  • x, y: the initial position of the entity
  • settings: a table containing physics related settings, all values are optional

Possible settings:

  • body_type: box2d body type: 'static', 'dynamic' or 'kinematic', defaults to 'dynamic'
  • shape: box2d shape name: 'rectangle', 'bsgrectangle', 'polygon', 'chain' or 'circle', defaults to 'rectangle'
  • vertices: a table of vertices to define the shape, used if the shape is either 'polygon' or 'chain', defaults to nil
  • loop: if the 'chain' shape should loop, defaults to false
  • w, h: width and height of the shape, used if the shape is either 'rectangle' or 'bsgrectangle', defaults to 32, 32
  • s: the corner cutting size in a 'bsgrectangle', defaults to 4
  • r: the radius of a 'circle', defaults to 16
  • collision_class: if this object should behave, in terms of collision ignoring (the .ignores class variable), as another class, defaults to the name of this object’s class 

:physicsBodyUpdate(dt[number])
  • dt: delta value passed from the main loop to update the body 

:physicsBodyDraw()
  • draws debug lines for the physics object if fg.debug_draw = true 

:addBody(area[Area], x[number], y[number], settings[table[any][optional]])
  • adds another body to this entity. Bodies/shapes/fixtures/sensors are stored in those attributes respectively; i.e. you can access all bodies via self.bodies[name]
  • area: a reference to the area where the entity is contained
  • x, y: the initial position of the entity
  • settings: a table containing physics related settings, all values are optional

The settings table follows the same specifications as in the constructor, however there’s one additional relevant possible value:

  • name: a string that is the name of this body/shape/fixture/sensor, which you can use to access those objects via self.bodies[name]. If omitted, defaults to 'main'. 

:addJoint(name[string], type[string], ...[multiple])
  • name: a string that is the name of this joint, which you can use to access it via self.joints[name]
  • type: a string specifying which joint type to add: 'distance', 'friction', 'gear', 'mouse', 'prismatic', 'pulley', 'revolute', 'rope', 'weld' or 'wheel'.
  • ...: multiple arguments that match the arguments required by the box2d joint creation call 

:changeCollisionClass(name[string], collision_class[string])
  • name: the name of the body to be changed
  • collision_class: the new collision class 

:removeBody(name[string])
  • name: the name of the body to be removed 

:removeJoint(name[string])
  • name: the name of the joint to be removed 

:removeShape(name[string])
  • name: the name of the shape to be removed

Attributes

.bodies[table[Body]]
  • a table containing physics bodies 

.body[Body]
  • the physics Body, alias for .bodies['main'] 

.fixture[Fixture]
  • the main Fixture, alias for .fixtures['main'] 

.fixtures[table[Fixture]]
  • a table containing fixtures 

.joints[table[Joint]]
  • a table containing joints 

.sensor[Fixture]
  • the secondary Fixture, used for generating callbacks when two objects physically ignore each other, alias for .sensors['main'] 

.sensors[table[Fixture]]
  • a table containing sensors 

.shape[Shape]
  • the physics Shape, alias for .shapes['main'] 

.shapes[table[Shape]]
  • a table containing shapes 

.shape_name[string]
  • the name of this object’s shape, possible values are 'rectangle', 'bsgrectangle', 'polygon', 'chain' and 'circle' 

.w[number], .h[number]
  • width and height, set if .shape_name is 'rectangle', 'bsgrectangle' or 'circle' 

.r[number]
  • radius, set if .shape_name is 'circle' 

.s[number]
  • corner cutting size, set if .shape_name is 'bsgrectangle'
