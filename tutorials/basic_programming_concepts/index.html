First Steps

This tutorial will teach you about the very basic concepts behind computer programming. If you know absolutely nothing about programming but you wanna make games, this is the right place to start! Before we actually start, though, you have to download a few things:

Text editor: this is what you use to create files containing commands that the computer runs. You could use Notepad if you wanted to, since it can create files, but there are text editors made just for programming that provide a lot of useful functionality. Two really good ones are: Notepad++ and Sublime Text.

Lua: Lua is a very beginner friendly language that is simple to understand, while being extremely powerful and expressible. If you use Windows, then you should install Lua for Windows. If you use OSX or Linux then you should already have Lua installed, and you can run its interpreter by typing lua in a terminal.

LÖVE: LÖVE is a 2D game making framework and it’s what you use to draw things to the screen, receive input from a keyboard/mouse/gamepad, play sounds, and so on… It’s also what Mogamett uses as the base for access to all of those functionalities. You can download it here.

This tutorial is divided in two parts. The first deals with basic programming concepts and uses only Lua. The second deals with applying those concepts into making a simple game, and uses both Lua and LÖVE. By the end of reading this and following along on your own computer (and also doing the exercises!!!), you should have the skills to make simple games using LÖVE. Anyway, are you ready? Then let’s go~!!!

Program Structure

The structure of a program in any language usually follows the same set of rules:

  • A file containing commands is given to the computer;
  • The computer runs each command contained on the file, line by line, in a top down manner;
  • The computer stops execution when it reaches the end of the file.

So, to see that happening, let’s first create a file. Open up your text editor and create a file named numbers.lua. In it, add the following commands (the numbers to the left side are line number markers, they’re not part of what you should add to the file):

1
2
3

print(1)
print(2)
print(3)

To tell the computer to run the file, we’re gonna use the Lua interpreter. You’ve installed it previously if you’re on Windows (Lua for Windows). Open a terminal (don’t know how to do it? Google it) and change to the directory (Google it) where you created the numbers.lua file. After that, type lua numbers.lua, and the numbers 1 2 3 should be printed to the screen, like in the image below:

What’s happening here is exactly as the rules previously stated said. A file is given to the computer and the computer runs it line by line. In this case, each line’s command is print, a function (I’ll use function interchangeably with command from now on) which simply prints to the screen whatever you tell it to print. We’re telling each print function to print one number, and so it does in the order we told it to. Each statement is independent of the other, such that if we remove print(2) then the output when you lua numbers.lua again will be 1 3.

But we can tell the print function to print other types of things, like an addition, or a subtraction, or a string (google it)! Add these to your numbers.lua file, run it and see the output. The numbers.lua file should look something like this:

1
2
3
4
5
6

print(1)
print(2)
print(3)
print(2+2)
print(6-1)
print('Hello~!')

Exercises
  1. What’s another way of running Lua programs?
    Answer
    Using the interactive Lua interpreter.
  2. What’s a string and why does it have to be enclosed in quotes?
    Answer
    A string is a sequence of characters (text data) and it has to be enclosed in some way so that the computer (more specifically the compiler) can tell that it is text data instead of being a program command.
  3. What’s the command line and how do you change directories using it?
    Answer
    Command line interpreters are programs that take commands and act on them, much like normal programs. The normal terminals that you have on Windows, OSX and Linux simply implement a bunch of useful functions related to the operating system at hand. Such a function is cd, which changes the current directory to the directory provided as an argument to it.


Variables

Computers are uncanilly good at remembering things. Unlike our mind’s memory, which works in a fuzzy associative way, a computer’s memory works more like a storage. Except that it’s a storage with perfect cataloging, cloning and teleporting powers: you know exactly where each package is and you can type in its address and press a button; the package is then cloned and the copy is instantly teleported to you. Addresses for this storage usually look something like this: 0x3A28213A, 0x6339392C, 0x7363682E, which are numbers using the hexadecimal numeral system.

Remembering those numbers every time you want to get a package from the storage is extremely tiresome and difficult, though. So what early storage people did was add the ability to refer to a certain address by a name that they picked: this is called a variable.

1
2
3

local a
a = 1
print(a)

Here a is a variable. The output of this piece of code, if you run it, is 1, and to understand why that is I’ll define two operations: load and store. Using the storage analogy, the load operation does the following: receive a handle/name, translate that handle into a storage address, clone the package contained in that address and teleport the copied package to a temporary location. The store operation does the following: receive a handle and a package, translate that handle into a storage address, teleport the package into the address, overwriting whatever other package was there.

So now we can understand, in terms of load and store, what’s going on with the code sample up there.

1

local a

The first line simply defines the a handle. This allocates a memory address that corresponds to this variable.

1

a = 1

The second line is a store operation. In programming, store operations use a single = sign. So a = 1 really means a receives 1 or 1 is stored in a. The proper way of expressing this operation in terms of store would be: store a, 1.

1

print(a)

The third line is the most complicated one. It does two things: a load operation and a function call. The computer (more specifically the compiler) knows that a is a programmer defined handle and that it corresponds to a certain address in memory, and it also knows that the print function wants the value contained in that address. So what it does is: load temp, a. Here temp is a temporary location in which the value 1 (the value contained in the address of the handle a) will be held - there are many temporary locations available and we can use them freely. After that, when the print function does its job, it will use temp to fetch the value of a. And so then the value 1 is printed to the screen.
