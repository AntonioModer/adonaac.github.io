lurker

A library that allows live code reloading (as the game is running), which can help with debugging as well as optimizing tweaking processes. It’s an exact copy of lurker, so refer to that for a list and an explanation of all functions it implements. It can be accessed through fg.lurker. fg.lurker_enabled is set to false by default because setting it to true makes it look for changes constantly, which depending on how fast your computer is can be pretty slow for big projects. Use something like fg.input:bind('f1', function() fg.lurker.scan() end) so that you can reload files whenever you press 'f1' instead of having lurker try to do it automatically all the time.

mlib

An utility library with tons of functions that makes operations regarding lines, polygons, circles, intersections and math related stuff in general a lot easier than they are with vanilla LÖVE/Lua. It’s an exact copy of mlib, so refer to that for a list and an explanation of all functions it implements. It can be accessed through fg.mlib.

Moses

An utility library with tons of functions that makes operations on tables, arrays, lists, collections and objects a lot easier than they are with vanilla Lua. It’s an exact copy of Moses, so refer to this tutorial for a list and an explanation of all functions it implements. It can be accessed through fg.moses, fg.mo or fg.fn.

utils

The fg.utils table holds some random utility functions…

Methods

.graphics.pushRotate(x[number], y[number], angle[number])
  • rotates everything around x, y by angle until love.graphics.pop() is called
  • x, y: the pivot position to rotate everything around
  • angle: the amount to rotate things for 

.graphics.pushRotateScale(x[number], y[number], angle[number], scale_x[number], scale_y[number])
  • rotates everything around x, y by angle with scale_x, scale_y until love.graphics.pop() is called
  • x, y: the pivot position to rotate everything around
  • angle: the amount to rotate things for
  • scale_x, scale_y: the amount to scale things by 

.math.chooseWithProbability(choices[table], chances[table])
  • returns a value from the choices table, chosen with a probability given by the chances table
  • choices: a table containing values to be picked
  • chances: a table of equal size to choices containing the probability of each value being picked

-- 50% to print 2
-- 30% to print 'hhh'
-- 20% to print 3
print(chooseWithProbability({2, 'hhh', 3}, {0.5, 0.3, 0.2})



.math.clamp(value[number], min[number], max[number])
  • clamps value between min and max 

.math.isBetween(value[number], min[number], max[number])
  • returns true if value is between min and max, false otherwise 

.math.random(min[number], max[number])
  • returns a random floating point number between min and max, use Lua’s math.random for integers only 

.math.round(number[number], precision[number])
  • returns a number rounded to precision decimal points
  • number: the number being rounded
  • precision: the precision, in terms of number of decimal places, that number is rounded to 

.table.copy(table[table])
  • returns the copied table. Since tables in Lua are always references this can be useful sometimes! 

.table.random(table[table])
  • returns a random value from table 

.angleToDirection4(angle[number])
  • returns a direction that best corresponds to angle, all possible returned values are 'right', 'up', 'left' and 'down' 

.directionToAngle4(direction[string])
  • returns an angle that corresponds to direction, all possible direction values are 'right', 'up', 'left' and 'down'
