-- prints 2 after 2 seconds from this call have passed
timer:after(2, function() print(2) end)

-- changes the value of some boolean every 1 second, but does it only 5 times
timer:every(1, function() some_boolean = not some_boolean end, 5)

-- prints 'SPAM' every frame for 5 seconds
timer:during(5, function() print('SPAM') end)

-- tweens the x, y position of self to 500, 500 for 2 seconds using the 'in-out-cubic' tween method
timer:tween(2, self, {x = 500, y = 500}, 'in-out-cubic')


Description

The fg.Timer module handles timers and tweens. It’s an enhanced version of hump.timer, with a few function name changes and with that added ability of timer/tween tagging. A global Timer is initialized automatically and can be used from fg.timer, but you may also create your own new instances by doing timer = fg.Timer(). Having your own instances is useful if some of your entities rely on tagging, since tags block timer/tween creation when a tag already exists, so using the global Timer for it is not going to work.

Timer/tween creation methods have tags enabled. Tags provide a way of identifying a certain tween so that it can be cancelled if necessary. For instance, this code:

function HPBar:changeHp()
    -- Cancel all HP bar related tweens
    self.timer:cancel('change_hp')
    self.timer:cancel('change_hp_back_after')
    self.timer:cancel('change_hp_back_tween')

    -- Tween the front layer of the HP bar
    self.timer:tween('change_hp', 0.2, self, 
                    {x_offset = self.w*(player.hp/player.max_hp)}, 'in-out-cubic')

    -- Tween the back layer (lighter color) of the HP bar after 0.2 seconds
    self.timer:after('change_hp_back_after', 0.2, function()
        self.timer:tween('change_hp_back_tween', 0.2, self, 
                        {x_offset_back = self.w*(player.hp/player.max_hp)}, 'in-out-cubic')
    end)
end

Does this:

The effect that the HP bar has of only actually decreasing after a while is achieved because of tags. The main tween 'change_hp' is done every time the player takes damage and it correponds to the front layer of the HP bar. Every time :changeHP is called, if another 'change_hp' tween is happening, it is cancelled and then immediately continued. If the :cancel call didn’t exist, there would be multiple tweens changing a single variable and it would look buggy.

The back layer, controlled by change_hp_back_after and 'change_hp_back_tween' happens after 0.2 seconds of the main tween. This explains why there’s a lag between taking damage and the HP bar actually decreasing. And similarly, these tweens are also cancelled on :changeHP call, this is to avoid the same visual bugs that would occur from multiple tweens acting on one variable. Additionally, this makes it so that the HP bar will only decrease after 0.2 seconds of the last damage received (since the 'change_hp_back_after' call is cancelled), giving it an even nicer lagging effect whenever the player is repeatedly damage in small bursts.

Methods

:new()
  • creates a new Timer instance 

:update(dt[number])
  • dt: delta value passed from the main loop to update the timer 

:after(tag[string[optional]], delay[number], action[function])
  • calls action after delay seconds, if tag is already registered then cancels previously registered after calls
  • tag: the identifier for this after call, the call is not tagged if this is omitted
  • delay: the amount of seconds from this call after which action will be called
  • action: the function to be called 

:cancel(tag[string])
  • cancels the timer/tween attached to tag 

:clear()
  • cancels all timers/tweens and removes all tags 

:destroy()
  • cancels all timers/tweens, removes all tags and dereferences the timer internal object 

:during(tag[string[optional]], duration[number], action[function], after[function[optional]])
  • calls action every frame for duration seconds, and then after after duration seconds, if tag is already registered then cancels previously registered during calls
  • tag: the identifier for this during call, the call is not tagged if this is omitted
  • duration: the amount of seconds action will be called for
  • action: the function to be called
  • after: the function to be called after duration seconds, optional and does nothing if omitted 

:every(tag[string[optional]], delay[number], action[function], count[number[optional]])
  • calls action every delay seconds for count times, if tag is already registered then cancels previously registered every calls
  • tag: the identifier for this every call, the call is not tagged if omitted
  • delay: the amount of seconds between each action call
  • action: the function to be called
  • count: the maximum amount of times action is called, defaults to infinite if omitted 

:tween(tag[string[optional]], duration[number], table[table], tween_table[table], tween_method[string], after[function[optional]])
  • tweens the attributes specified in tween_table from table to the values specified in tween_table using tween_method, calls after when the tween is done
  • tag: the identifier for this tween call, the call is not tagged if omitted
  • duration: the duration in seconds of the tween
  • table: the table that is being tweened
  • tween_table: the values specifying which attributes and to what values those attributes will be changed
  • tween_method: the tween method used for the tween
  • after: the function to be called after duration seconds, optional and does nothing if omitted

Refer to hump.timertween for more information.
