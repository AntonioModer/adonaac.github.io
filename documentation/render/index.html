Render

The render module takes care of drawing everything in the engine. It’s attached to the fg.world instance and so all its functions should be called from there. It provides access to a camera, as well as layers to which you can add objects and then sort their draw order, add layer/screen-wide shaders, etc, etc.

Layer

Layers are useful for separating objects into groups and then sorting those objects/groups in whatever way necessary in your game. The render module has a list of layers and offers a few functions to interface with them. The funcionality for now is limited to drawing orders (object draw order inside a layer and draw order of layers in relation to one another) and shaders (apply a layer/screen-wide shader to a certain layer). Each class/object needs to have its layer defined, otherwise the 'Default' one is picked. If defined, fuccboiGDX automatically adds new objects to the appropriate layer. A layer can be set as a class variable or as an object attribute, the latter having preference if both are defined.

Layer Methods

:addLayer(layer_name[string], settings[table[any][optional]])
  • adds a layer to the world, a layer named 'Default' is added by default and all objects are placed in it if they (or their class) have no layer attribute defined
  • layer_name: the name of the layer to be added
  • settings: a table with settings for the layer

Possible settings:

  • not_sortable: makes it so that the objects inside this layer can’t be sorted when :sortRenderOrder is called
  • parallax_scale: how close or far away this layer is from the camera, 1 is the normal value, <1 is closer >1 is further away 

:addShaderToLayer(layer_name[string], shader_name[string], shader_vertex_path[string], shader_fragment_path[string])
  • adds a shader with the identifier shader_name to the layer named layer_name. The canvas created by this layer (to which all its objects get drawn to) will then be affected by this shader.
  • layer_name: the name of the layer the shader is being added to
  • shader_name: the name of the shader to be added
  • shader_vertex_path: the path to the vertex shader
  • shader_fragment_path: the path to the fragment shader
1
2

fg.world:addLayer('Blood_Floor')
fg.world:addShaderToLayer('Blood_Floor', 'Outline', 'resources/shaders/default.vert', 'resources/shaders/outline.frag')



:addToLayer(layer_name[string], entity[Entity])
  • adds entity to the layer named layer_name, used internally most of the time but useful for adding spritebatches or backgrounds when needed
  • layer_name: the name of the layer where the entity will be added
  • entity: the entity to be added 

:sendToShader(layer_name[string], shader_name[variable], string[value], any[])
  • sends the variable, value pair to the shader, see Shader:send for more information
  • layer_name: the name of the layer being affected by this value
  • shader_name: the name of the shader on that layer being affected by this value
  • variable: the name of the variable
  • value: the value of the variable
1
2

-- in some update function
fg.world:sendToShader('Blood_Floor', 'Outline', 'brightness', 0.4)



:setShaderClassList(layer_name[string], shader_name[string], class_list[table[string]])
  • makes it so that the classes defined in list are the only ones in this layer affected by this shader
  • layer_name: the name of the layer
  • shader_name: the name of the shader
  • class_list: a list of class names defining which classes get affected by this shader on this layer 

:setLayerOrder(layers_order[table[string]])
  • sets the rendering order for all layers
  • layers_order: the layer order table, first layers are drawn first, last are drawn last
1

fg.world:setLayerOrder({'Back', 'Middle', 'Default', 'Front', 'Effects'})



:sortLayerRenderOrder(layer_name[string], order_function[function])
  • sorts all objects in the layer named layer_name according to order_function
  • layer_name: the name of the layer being sorted
  • order_function: the order function, see table.sort for more information 

:sortRenderOrder(order_function[function])
  • sorts all objects in the all layers according to order_function
  • order_function: the order function, see table.sort for more information
1
2
3

-- sorts all objects according to their .y attribute, useful for games in a Zelda-ish topdown angle
-- in some update function
fg.world:sortRenderOrder(function(a, b) return a.y < b.y end)


Render Methods

:renderAttach()
  • attaches fg.world.camera, applying all its transformations until :renderDetach() is called 

:renderDetach()
  • detaches the camera, removing all its transformations
