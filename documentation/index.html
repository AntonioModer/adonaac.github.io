fuccboiGDX is composed of two parts: the framework and the engine.

The framework contains some user created LÃ–VE libraries that are useful for game development. Most libraries chosen are simple, self-contained and composable so that they can be used independently of each other with few problems. This is in a similar fashion to how LÃ–VEâ€™s own modules work.

The engine uses modules from the framework as well as a bunch of glue code to build a higher level programming experience at the cost of customizability and control. The features provided by the engine have the goal of making the programmerâ€™s life easier, taking care of some tedious work usually goes into making a game, things like: box2d wrappers, collision handling, map saving/loading, levels, pooling, and so onâ€¦

The words framework and engine will be used throughout the documentation and usually you only have to pay more attention when the engine is mentioned, since it implies that things are tied together in sometimes initially unintuitive or unexpected ways. Modules

Main

  • fg - core, contains everything
  • Render - orchestrates all engine rendering functionality (inside the fg.world instance)
  • World - orchestrates all engine functionality, an instance named fg.world is automatically created on initialization

Logic

  • Area - an area/level/map where objects reside in
  • Class - object orientation library, rxi/classic
  • Collision - internal module used for handling box2d collisions, read to understand how collisions are handled
  • Debug - browser-based debug console for LÃ–VE, rxi/lovebird
  • Entity - base class for all engine game objects
  • Gamestate - game state system, vrld/hump.gamestate
  • Input - binds keys to actions, allows for pressed/released checks outside of LÃ–VEâ€™s callbacks and supports gamepads
  • Loader - threaded resource loader, kikito/love-loader
  • ObjectPool - object pooling for higher performance on objects that are killed/created often
  • PhysicsBody - mixin used for adding physics capabilities to an object
  • Serial - serialization for areas, objects and tables, uses gvx/Ser internally
  • Solid - the base collision solid entity
  • Timer - timers, tweens and allows timer/tween tagging, an enhanced version of vrld/hump.timer
  • Utils - random utility functions, Yonaba/Moses, davisdude/mlib
  • Vector - 2D vector class, vrld/hump.vector

Visual

  • Animation - creates animated sprites from sprite sheet images, an enhanced version of bartbes/AnAL
  • Background - backgrounds for use with parallaxed layers
  • Camera - similar to HaxeFlixelâ€™s camera with a few improvements, an enhanced version of vrld/hump.camera
  • Particles (Engine) - a simple particle systems module, also has an editor for editing particle systems
  • Spritebatch - batch draw calls from a single image for more performant draw code
  • Text - programmable text drawing
  • Tilemap - tilemaps with autotiling, collision solid generation and Tiled support
  • UI Tools - UI module for building tools, KennyShields/loveframes

Sound

  • Sound - sound management system, (Ensayia & Taehl)/TEsound
