Description

An object pooling module that is kinda based on ImpactJS’ entity pooling system. Both those links do a good job of explaining why pooling is necessary and how it works.

fuccboiGDX’s pooling system needs two things so that it can work: on an engine class (one that inherits from Entity) a .pool_enabled class attribute with the size of the pool to be created for that class; and a :reset function that is called every time the object created needs to be reused. This call will usually do the same things as the constructor does, and it receives as arguments the x, y values for its position as well as the settings table.

MyClass = fg.Class('MyClass', 'Entity')

MyClass.pool_enabled = 100
...

function MyClass:reset(x, y, settings)
    ...
end

When an object is killed (via self.dead = true), it goes back to the pool until it gets selected again, and that’s when the :reset method is called. If there are no free objects in a pool then by default you won’t be able to create new objects until another one dies and goes back to the pool to be reused, but there are ways of specifying different overflow behaviors through the .pool_overflow_rule class variable.

MyClass = fg.Class('MyClass', 'Entity')

MyClass.pool_enabled = 100
MyClass.pool_overflow_rule = 'distance'
...

function MyClass:reset(x, y, settings)
    ...
end

The possible values for the .pool_overflow_rule class variable are: 'oldest', 'distance', 'random' and 'never'. 'oldest' picks the oldest object created to be reallocated, 'distance' picks the object further away from the center of the screen (more likely to be out of view) to be reallocated, 'random' picks a random one and 'never' is the same as the default behavior.
